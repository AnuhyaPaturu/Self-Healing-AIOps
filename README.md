ğŸ—ï¸ Technical Architecture
The system follows a standard OODA loop (Observe, Orient, Decide, Act) architecture:

Observe: Metrics are scraped from the host machine using Prometheus.

Orient: Data is processed in Python and normalized for Machine Learning.

Decide: The Isolation Forest algorithm calculates an anomaly score and a confidence percentage.

Act: The Remediation Engine (psutil) executes a precision kill of the offending process.

ğŸ§  The "Why" (Engineering Decisions)
Why Isolation Forest? Unlike simple "If CPU > 90%" rules, this AI learns your unique system baseline. It can detect a "frozen" app at 30% CPU if that behavior is statistically abnormal for your environment.

Explainable AI (XAI): The dashboard displays a Confidence Score, moving the system from a "Black Box" to a transparent, trustworthy tool.

Safety Gate: Implementation of a Human-in-the-Loop threshold ensures that the AI doesn't act on "false positives," protecting system stability.
